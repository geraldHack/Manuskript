Agenten im Kapitel-Editor

Das Agenten-Panel rechts im Editor bündelt KI-gestützte Analyse, Szenen-Schreiben, Chat und Überarbeiten. Es ersetzt den früheren separaten KI-Assistenten.

Panel öffnen und schließen

  • Toolbar-Button Agenten (Toggle) – Panel ein-/ausblenden
  • Während Online-Lektorat läuft, wird das Agenten-Panel durch das Lektorat-Panel ersetzt
  • Oben erscheint ein Activity-Banner, wenn ein Agent arbeitet

Agenten-Typen

  • Analyse – Plothole, Dialog, Textstruktur, Show-don't-tell usw.; findet Probleme mit Zitat und Verbesserungsvorschlägen
  • Szene schreiben – neue Szenen anhand von Anweisung und Szenen-Kontext (Outline, vorherige Kapitel)
  • Chatbot – Dialog mit KI unter Einbezug von Projektkontext (Weltdateien, Kapitel)
  • Überarbeiten – markierten Text umschreiben (auch per Kontextmenü „Überarbeiten“)

Workflow (Analyse)

  1. Agent-Tab wählen (z. B. Plothole-Agent)
  2. Optional: Anweisung anpassen
  3. Analyse starten – der Agent liest den Kapiteltext (ggf. mit Kontext aus anderen Kapiteln)
  4. Befunde erscheinen im Text markiert; Vorschläge im Panel
  5. Vorschlag übernehmen oder ablehnen

Konfiguration pro Tab

Über Konfiguration (ausklappbar) pro Agent-Tab:

  • System-Prompt und Default-Prompt (Anweisung)
  • Backend: Ollama oder OpenAI (pro Agent in config/agents.json)
  • Sampling: Temperatur, Max Tokens, Top-P, Repeat Penalty
  • Bei Szene-Agent: Default-Prompt wird in Anweisungsfeld übernommen; Szenen-Kontext-Größe wählbar

Neue Agenten

  • Tab + – neuen Agent anlegen (Typ wählen)
  • Definitionen werden in config/agents.json gespeichert

Backends und Parameter

Ollama (lokal): agent.ollama.api_url, agent.ollama.model – Ollama muss laufen (ollama serve).

OpenAI-kompatibel: agent.openai.api_key, agent.openai.api_url, agent.openai.model.

Globaler Standard: agent.backend (Ollama oder OpenAI). Ein-/Aus: agent.enabled.

Echtzeit-Analyse

Optional (agent.realtime_enabled): Analyse-Agenten können nach längerer Schreibpause automatisch erneut laufen (Debounce über agent.realtime_debounce_ms).

Hinweis: Cloud-Backends (OpenAI) erfordern API-Key und können Kosten verursachen. Ollama läuft lokal, benötigt aber ausreichend RAM/GPU für größere Modelle.
