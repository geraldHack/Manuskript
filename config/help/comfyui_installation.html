🎧 ComfyUI Installation für Manuskript

Kostenlose lokale Sprachsynthese mit Qwen3-TTS

Bereit für Installation

Windows Installation (empfohlen)

Schritt 1: ComfyUI herunterladen

Laden Sie die offizielle ComfyUI-Version für Windows:

📥 ComfyUI für Windows herunterladen

Nach dem Download: Doppelklick auf die .exe-Datei und Installation folgen.

Schritt 2: Python prüfen (falls nicht vorhanden)

ComfyUI benötigt Python 3.10+. Prüfen Sie ob Python installiert ist:

python --version

Falls nicht: Python herunterladen

⚠️ Wichtig: Bei Python-Installation "Add Python to PATH" aktivieren!

Schritt 3: ComfyUI starten

Nach der Installation finden Sie ComfyUI im Startmenü:

  1. Startmenü → "ComfyUI" öffnen
  2. Warten Sie bis "Starting server on 127.0.0.1:8188" erscheint
  3. ComfyUI bleibt im Hintergrund laufen

Schritt 4: Qwen3-TTS installieren

Öffnen Sie die Eingabeaufforderung (Windows-Taste + R, dann "cmd" eingeben) und führen Sie aus:

cd %USERPROFILE%\ComfyUI\custom_nodes
git clone https://github.com/flybirdxx/ComfyUI-Qwen-TTS
cd ComfyUI-Qwen-TTS
pip install -r requirements.txt
⚠️ Danach ComfyUI neu starten, damit die neuen Nodes geladen werden (siehe Abschnitt „ComfyUI neu starten in Manuskript“ unten).

Schritt 5: Qwen3-Modell downloaden

Wichtig: Für HuggingFace benötigen Sie einen kostenlosen Account!

  1. Besuchen Sie huggingface.co/join und registrieren Sie sich
  2. Loggen Sie sich ein
  3. Laden Sie ein Qwen3-TTS Modell (empfohlen: 1.7B Base):
📥 Qwen3-TTS 1.7B Base (empfohlen) 📥 Qwen3-TTS 0.6B Base (kleiner)

Nach dem Download entpacken und nach %USERPROFILE%\ComfyUI\models\Qwen3-TTS verschieben

Schritt 6: In Manuskript testen

  1. Manuskript starten
  2. Kapitel wählen → Toolbar „Sprachsynthese“
  3. Eine ComfyUI-Stimme wählen (kein ElevenLabs)
  4. Text markieren und „Erstellen“ klicken
  5. Bei Fragen: ?-Button neben der Stimmen-Auswahl (öffnet diese Hilfe)
✅ Wenn Audio abgespielt wird, ist alles eingerichtet!

macOS Installation

Schritt 1: ComfyUI herunterladen

Für Apple Silicon (M1/M2/M3):

📥 ComfyUI für macOS (Apple Silicon) herunterladen

Nach dem Download: DMG-Datei öffnen und ComfyUI in den Anwendungsordner ziehen.

Schritt 2: Homebrew installieren (falls nicht vorhanden)

Homebrew ist der Paketmanager für macOS:

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Schritt 3: Python und Git installieren

brew install python@3.10 git

Schritt 4: ComfyUI starten

ComfyUI aus dem Programme-Ordner starten (DMG-Installation) oder im Terminal:

cd /Applications/ComfyUI
python3 main.py

Warten bis Starting server on 127.0.0.1:8188 erscheint. Der ComfyUI-Ordner kann je nach Installation abweichen – wichtig ist der Ordner mit Unterverzeichnis custom_nodes.

Schritt 5: Qwen3-TTS installieren

Terminal öffnen (ComfyUI kann laufen oder gestoppt sein) und ausführen – Pfad ggf. anpassen:

cd /Applications/ComfyUI/custom_nodes
git clone https://github.com/flybirdxx/ComfyUI-Qwen-TTS
cd ComfyUI-Qwen-TTS
pip3 install -r requirements.txt
⚠️ Danach ComfyUI neu starten, damit die neuen Nodes geladen werden (siehe Abschnitt „ComfyUI neu starten in Manuskript“ unten).

Schritt 6: Qwen3-Modell downloaden

Wichtig: Für HuggingFace benötigen Sie einen kostenlosen Account!

  1. Besuchen Sie huggingface.co/join und registrieren Sie sich
  2. Loggen Sie sich ein
  3. Laden Sie ein Qwen3-TTS-Modell herunter (empfohlen: 1.7B Base für „Hohe Qualität“ in Manuskript):
📥 Qwen3-TTS 1.7B Base (empfohlen) 📥 Qwen3-TTS 0.6B Base (kleiner, schneller)

Auf HuggingFace: Files and versions → alle Dateien des Modells herunterladen (oder per git lfs clone).

Entpacken und in den Ordner legen:

/Applications/ComfyUI/models/Qwen3-TTS/

Der Ordner muss existieren (mkdir -p /Applications/ComfyUI/models/Qwen3-TTS). Die Modell-Dateien müssen so liegen, wie es der Node ComfyUI-Qwen-TTS erwartet (siehe README des Repos).

Schritt 7: ComfyUI neu starten

Nach Schritt 5 und 6 ComfyUI beenden und erneut starten (Schritt 4), oder den Button „ComfyUI neu starten“ in Manuskript nutzen (siehe Abschnitt unten).

Schritt 8: In Manuskript testen

  1. Manuskript starten
  2. Kapitel wählen → Toolbar „Sprachsynthese“
  3. Eine ComfyUI-Stimme wählen (kein ElevenLabs)
  4. Text markieren und „Erstellen“ klicken
  5. Bei Fragen: ?-Button neben der Stimmen-Auswahl (öffnet diese Hilfe)
✅ Wenn Audio abgespielt wird, ist alles eingerichtet!

🔄 ComfyUI neu starten in Manuskript

Wann nötig?

  • Nach Installation oder Update von ComfyUI-Qwen-TTS (Custom Nodes)
  • Wenn Manuskript meldet, dass Nodes fehlen oder TTS fehlschlägt
  • Nach dem Kopieren neuer Modell-Dateien (manchmal ausreichend; bei Problemen neu starten)

Manuskript und ComfyUI sind getrennte Programme – ein Neustart von ComfyUI beendet Manuskript nicht.

Button „ComfyUI neu starten“

Im Fenster Sprachsynthese (Toolbar → Sprachsynthese) erscheint in der Statuszeile der Button „ComfyUI neu starten“, sobald eine ComfyUI-Stimme (nicht ElevenLabs) gewählt ist.

Der Button ist nur aktiv, wenn in den Manuskript-Parametern ein Restart-Skript hinterlegt ist:

  1. Parameter-Verwaltung öffnen
  2. Kategorie TTS (ComfyUI)
  3. comfyui.restart_script → Pfad zu einem Skript, das ComfyUI beendet und neu startet
⚠️ Ist kein Skript eingetragen, bleibt der Button ausgegraut – dann ComfyUI manuell neu starten (Terminal/App schließen und Schritt „ComfyUI starten“ wiederholen).

Beispiel macOS (~/bin/restart-comfyui.sh, ausführbar mit chmod +x):

#!/bin/bash
pkill -f "ComfyUI" 2>/dev/null || true
sleep 2
cd /Applications/ComfyUI && python3 main.py &

Pfad in comfyui.restart_script eintragen, z. B. /Users/IhrName/bin/restart-comfyui.sh.

Alternative in der ComfyUI-Oberfläche

In ComfyUI selbst gibt es oft Reload custom nodes – das reicht manchmal nach Node-Updates. Wenn TTS in Manuskript weiterhin fehlschlägt, ComfyUI vollständig neu starten.

🆘 Häufige Probleme & Lösungen

"ComfyUI nicht erreichbar"

  • ComfyUI muss im Hintergrund laufen
  • URL prüfen: http://127.0.0.1:8188
  • Firewall könnte blockieren

"Python nicht gefunden"

  • Python neu installieren mit "Add to PATH"
  • System neu starten

"CUDA out of memory" / Speicher voll (GPU)

  • ComfyUI mit CPU starten: python main.py --cpu (bzw. python3 auf macOS/Linux)
  • oder kleineres Modell verwenden (0.6B statt 1.7B)

„ComfyUI neu starten“ ausgegraut

  • In den Parametern comfyui.restart_script auf ein existierendes Skript setzen
  • Oder ComfyUI manuell neu starten

Qwen3-Nodes fehlen / unbekannter Node-Typ

  • Prüfen: custom_nodes/ComfyUI-Qwen-TTS von flybirdxx installiert?
  • ComfyUI danach neu starten (nicht nur Browser-Tab neu laden)

✅ Installations-Checkliste

  • ComfyUI heruntergeladen und installiert
  • Python 3.10+ verfügbar
  • ComfyUI startet und zeigt "127.0.0.1:8188"
  • Qwen3-Nodes installiert
  • Qwen3-Modell heruntergeladen
  • Manuskript kann auf ComfyUI verbinden
  • ComfyUI nach Node-Installation neu gestartet
  • Optional: Restart-Skript in Manuskript-Parametern hinterlegt
  • Erste Audio-Generierung erfolgreich
